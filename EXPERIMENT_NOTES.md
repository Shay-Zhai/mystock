# 历史实验结论

> 本文档记录策略迭代过程中的关键实验与结论，供后续优化参考，避免重复踩坑。

## 一、交易执行方式实验

### 实验1.1：T+1开盘价执行 vs 收盘价执行+滑点
**背景**：原方案在调仓日收盘价计算策略，次日开盘价执行。

**结果**：
- 次日开盘价执行收益明显降低
- 改为收盘价执行 + 双边各加0.1%滑点（共0.2%+手续费0.05%×2=0.3%总成本）后，收益更贴近实际

**结论**：**采用收盘价交易+滑点模式**。回测中`buy_cost_rate = sell_cost_rate = 0.0005 + 0.001 = 0.0015`

---

## 二、确定性Bug实验

### 实验2.1：策略结果不确定性
**现象**：`use_multi_momentum`从True→False→True切换后，两次True的年化收益不同（13.73% vs 13.98%）

**根因**：Python的set遍历顺序受`PYTHONHASHSEED`影响随机化，当现金不足时买入顺序不同导致持仓差异

**修复**：所有set遍历改为`sorted()`，确保交易顺序确定性

**结论**：**永远不要在交易逻辑中直接遍历set**，必须先`sorted()`

## 三、止损策略实验（核心）

### 实验3.1：止损模式对比
**四种模式表现**（多期限动量策略，2018-2025）：

| 模式 | 年化收益 | 最大回撤 | 夏普 | 止损次数 |
|------|---------|---------|------|---------|
| 无止损 | 14.50% | -13.88% | 0.30 | 0 |
| 仅单票止损(0.5×60日σ) | 1.97% | -9.94% | - | 频繁 |
| 仅整体止损(12%/18%) | 4.09% | -30.94% | 0.10 | 5 |
| 单票+整体 | 1.97% | -9.94% | - | 频繁 |

### 实验3.2：单票止损阈值优化
原方案`0.5 × 60日收盘价标准差`阈值过紧（收盘价标准差本身很小）：

| 单票止损方式 | 年化 | 回撤 | 止损次数 |
|------------|------|------|---------|
| 0.5×60日σ（原方案） | 1.97% | -9.94% | 极频繁 |
| 10%百分比回撤 | 11.19% | -15.87% | 16 |
| 12%百分比回撤 | 11.51% | -14.57% | 11 |
| **15%百分比回撤** | **13.00%** | **-14.35%** | **6** |

**结论**：**单票止损采用15%百分比回撤**，仅在极端情况下触发，几乎不影响策略表现

### 实验3.3：整体止损的反效果
**关键发现**：整体止损在动量轮动策略中有害

- 整体止损(18%减半/25%清仓)：年化6.53%，回撤-27.87%（vs 无止损14.50%/-13.88%）
- 即使将清仓阈值提高到25%（取消减半），回撤仍恶化到-27.87%

**根因**：动量策略在市场大跌后清仓，下一个调仓日（20交易日后）重新满仓，形成"底部卖出、反弹后买回"的高买低卖循环。整体止损与动量策略"持有趋势"的核心逻辑冲突。

**最终方案**：**默认仅启用单票止损（`mode='individual'`），整体止损参数保留但默认不启用**

```python
'stop_loss': {
    'mode': 'individual',                # 当前默认
    'individual_drawdown_pct': 0.15,     # 单票15%回撤止损
    'portfolio_dd_half': 0.18,           # 保留参数
    'portfolio_dd_clear': 0.25,          # 保留参数
}
```

### 实验3.4：整体止损Bug修复记录
- **Bug1**：12%减半止损每日重复触发，几乎清空持仓 → 添加`portfolio_half_sold`标志，每次回撤事件仅触发一次，回撤恢复到阈值下方时重置
- **Bug2**：缺少`sl_mode`检查，导致`none`模式也触发止损 → 添加`if sl_mode in ('portfolio', 'both')`条件

## 四、动量策略参数实验

### 实验4.1：6-1跳月动量初始实验
**背景**：用户希望适配经典6-1动量（6个月动量+跳1月+200日均线）

**初始方案**（126日动量+跳21日+200日均线）：年化2.80%，回撤-42.25%

**诊断**：200日均线是主要问题

### 实验4.2：趋势均线周期敏感性
6-1动量（126日跳21日）+ 不同趋势均线：

| 趋势均线 | 年化收益 | 最大回撤 | 夏普 |
|---------|---------|---------|------|
| 200日 | 2.27% | -43.01% | -0.02 |
| 100日 | 5.57% | -35.16% | 0.07 |
| 50日 | 4.82% | -35.01% | 0.05 |
| **20日** | **15.34%** | -25.65% | 0.36 |

**根因**：200日均线在熊市中过滤掉所有ETF，策略空仓；市场反弹越过200日均线时，126日动量仍为负值，无法重新建仓——形成"底部空仓、踏空反弹"循环

**结论**：**采用20日均线**过滤

### 实验4.3：跳过期数敏感性（关键发现）
126日动量 + 20日均线 + 不同跳过期数：

| 跳过期数 | 年化收益 | 最大回撤 | 夏普 | 卡玛 |
|---------|---------|---------|------|------|
| 不跳(0日) | 11.69% | -14.83% | 0.25 | 0.79 |
| **跳10日** | **15.66%** | **-14.30%** | **0.39** | **1.10** |
| 跳21日 | 15.34% | -25.65% | 0.36 | 0.60 |
| 跳42日 | 3.79% | -59.76% | 0.02 | 0.06 |

**关键发现**：**跳10日（2周）是最优组合**，全面超越其他方案
- 跳21日虽然年化接近，但回撤放大近一倍
- 跳42日完全失效

**原因推测**：A股ETF短期反转效应约2周，跳10日正好规避；跳21日损失过多时效性信号，趋势转折点反应迟缓

### 实验4.4：动量回看周期敏感性
跳21日 + 200日均线 + 不同动量周期：

| 动量周期 | 年化收益 | 最大回撤 |
|---------|---------|---------|
| 60日 | 7.80% | -32.63% |
| 126日 | 2.27% | -43.01% |
| 252日 | 3.04% | -43.67% |

**结论**：过长的动量周期（252日）信号滞后严重，失效

### 实验4.5：持仓数量敏感性
126日跳21日 + 20日均线 + 不同持仓数：

| 持仓数 | 年化收益 | 最大回撤 |
|-------|---------|---------|
| 3只 | 10.62% | -16.69% |
| 6只 | 15.34% | -25.65% |
| 10只 | 10.20% | -25.69% |

**结论**：6只持仓最优，过多或过少均不佳

## 六、方法论总结

### 6.1 有效做法
1. **多参数对比分析**：每次策略调整都生成对比表，避免单点测试的偶然性
2. **sorted()保证确定性**：所有set遍历必须排序
3. **止损参数保留可切换**：四种模式通过`mode`字段切换，便于对比
4. **保留多套策略配置**：`SIMPLE_STRATEGY_CONFIG`、`SKIP_MOMENTUM_STRATEGY_CONFIG`等，便于切换

### 6.2 失败做法
1. **整体止损+动量策略**：动量策略的"持有趋势"逻辑与"底部清仓"冲突
2. **200日均线过滤**：在A股熊市后踏空反弹
3. **基于价格标准差的止损**：收盘价标准差本身很小，0.5倍阈值过紧
4. **过长的动量周期**（252日）或**过长的跳过期数**（42日）：信号严重滞后

### 6.3 待探索方向
- ATR波动率止损（2-3倍ATR）替代百分比止损
- 不同调仓周期（10日/20日/40日）的对比
- 多策略组合（多期限动量 + 6月跳月动量）
- 行业轮动与宽基轮动的分离测试
- 加入估值因子（PE/PB）作为辅助过滤

## 七、临时调试脚本规范
实验性脚本以`_`开头命名（如`_test_stoploss.py`、`_compare_strategies.py`），实验完成后删除。避免长期堆积临时文件。

对比脚本模板：
```python
"""对比X与Y"""
import pandas as pd
from data import get_multiple_etf_hist
from strategy import SimpleMomentumStrategy
from backtest import Backtest
from stock import ETF_POOL, ETF_NAMES, BACKTEST_CONFIG

etf_codes = []
for category in ETF_POOL.values():
    etf_codes.extend(category)
price_data_dict = get_multiple_etf_hist(
    etf_codes, BACKTEST_CONFIG['start_date'], BACKTEST_CONFIG['end_date'], min_length=50)

def run_test(config, label):
    strategy = SimpleMomentumStrategy(**config)
    bt = Backtest(initial_capital=BACKTEST_CONFIG['initial_capital'],
                  rebalance_cost=BACKTEST_CONFIG['rebalance_cost'],
                  slippage=BACKTEST_CONFIG.get('slippage', 0.001),
                  stop_loss_config={'mode': 'none'})
    m = bt.run(price_data_dict, strategy,
               pd.to_datetime(BACKTEST_CONFIG['start_date']),
               pd.to_datetime(BACKTEST_CONFIG['end_date']),
               rebalance_period=BACKTEST_CONFIG['rebalance_period'],
               etf_names=ETF_NAMES)
    print(f"{label:<30} 年化={m['annual_return']*100:>6.2f}%  回撤={m['max_drawdown']*100:>6.2f}%  夏普={m['sharpe_ratio']:>5.2f}")
    return m

print(f"{'配置':<30} {'年化':>8} {'回撤':>8} {'夏普':>6}")
print("-" * 60)
run_test({...}, "标签")
```
